The Scheduling Subsystem simulator operates in a number of distinct modes.

  • Full simulation
    • MasterScheduler, ProjectManager, and Schedulers are all running.
    • Full range of project manager actions are performed.
    • External responses are generated.
    • Full report of internal activities is generated.
  • Planning mode
    • MasterScheduler and Schedulers in dynamic mode are running.
    • ProjectManager does not run.
    • No external responses are generated.
    • The order in which blocks are executed is reported.
  • Fixed event mode
    • This is a special purpose class that only engages in conflict detection. Only fixed events and blocks that must be scheduled at a particular time are dealt with. Its purpose is to detect any conflicts in scheduling these events.

The requirements for the first phase of planning mode are as follows.

  • The PipelineSimulator and CalibrationSimulator are null.
  • The ArchiveSimulator must be able to read scheduling blocks and project definitions from many sources.
  • The ExcecutiveSimulator will start the MasterScheduler, read all events and generate the reports that define the schedule.
  • The ControlSimulator functions in the normal manner.
