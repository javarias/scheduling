public class ALMAArchive

The ArchiveProxy class is the interface to the real or simulated archive.

public class ALMAClock extends ClockBase

The Clock class gives access to the array time, as implemented by the Control System, in the real mode. In the simulation mode, this class accesses the ClockSimulator in the simulation class.

public class ALMADispatcher implements ControlProxy

The Dispatcher class presents a list of scheduling blocks for execution to the telescope operator, in priority order, and waits for a selection. If no selection is made, then the first one is selected. It then initiates the execution of the selected scheduling block.

public interface ControlProxy

The ControlProxy interface specifies those methods needed by Scheduling from the Control system. The methods are implemented by the ALMADispatcher and by the ControlSimulator.

public class MasterSBQueue

The MasterSBQueue class is the collection of scheduling blocks currently under consideration by the scheduling system.

public class MasterSchedulerAction implements Runnable

The MasterSchedulerAction class encapsulates periodic actions that must be executed by the MasterScheduler. these include, getting new scheduling blocks and project definitions from the archive, analyzing the master queue for fixed events and possible changes in sub-array configuratiions.

public interface Scheduling_to_TelescopeOperator

This interface is implemented in the TelescopeOperator class using interfaces in the Executive subsystem. (See ICD sectio 3.4) This interface is used by the MasterScheduler and Schedulers to send information to the telescope operator. In some cases replies are expected.

public class TaskControlInfo

TaskControlInfo is an informational object that is used by the MasterScheduler to monitor and control a subordinate component that is a thread. Such an object is shared between the MasterScheduler and the subordinate thread.
